# Generate Weekly Briefing

Audit the past week's activity and generate a "Monday Morning CEO Briefing" in `/Briefings/`.

## When to Use

- Every Monday morning (triggered by Windows Task Scheduler / cron)
- When the user asks for a business summary
- At end-of-week review

## Steps

### 1. Read Context Files

Read all of these:
- `Business_Goals.md` — targets, KPIs, alert thresholds
- `Dashboard.md` — current system state
- `Company_Handbook.md` — business context, subscription audit rules

### 2. Gather Week's Data

Scan these folders and files for the past 7 days:

**Completed items:**
- Count files in `/Done/` created this week
- Note any plans marked `status: completed` in `/Plans/`

**Log summary:**
- Read all `/Logs/{date}.json` files from the past 7 days
- Count by `action_type`: emails_processed, linkedin_post_published, etc.
- Identify any errors (action_type containing "error" or "failed")

**Pending items:**
- Count files in `/Needs_Action/` still `status: pending`
- Count files in `/Pending_Approval/` still pending

**Plans:**
- Review `/Plans/` for in-progress plans
- Note any overdue plans (created > 3 days ago, still `status: in_progress`)

### 3. Generate the Briefing

Create `/Briefings/{YYYY-MM-DD}_Monday_Briefing.md`:

```markdown
---
generated: {ISO timestamp}
period: {start_date} to {end_date}
generated_by: claude_code
tier: silver
---

# Monday Morning CEO Briefing
## Week of {start_date} – {end_date}

---

## Executive Summary

{2-3 sentences: overall health of the business this week. Be specific.}

---

## System Activity This Week

| Metric | Count |
|--------|-------|
| Emails processed | {N} |
| Emails replied (approved) | {N} |
| LinkedIn posts published | {N} |
| Files triaged | {N} |
| Plans completed | {N} |
| Items in /Done | {N} |
| Errors logged | {N} |

---

## Completed This Week

{List each item moved to /Done/ this week:}
- [{date}] {item description}

---

## Active Plans (In Progress)

{List each plan from /Plans/ still in_progress:}
- [{plan filename}] {objective} — created {date}
  {Flag if overdue (> 3 days old)}

---

## Pending Items Requiring Attention

| Item | Age | Priority | Status |
|------|-----|----------|--------|
{For each file in /Needs_Action/ or /Pending_Approval/:}
| {filename} | {X days} | {priority} | {status} |

---

## Proactive Suggestions

{Based on Business_Goals.md thresholds and log data:}

### Revenue
- **MTD Progress:** {current MTD} / {monthly goal} ({%} of target)
- {Flag if below pace with specific recommendation}

### Bottlenecks
{Any plans or items stuck > 3 days — specific actionable note}

### Cost Optimization
{Check for subscriptions per Company_Handbook.md §Subscription Audit Rules}
- {Flag any that haven't been touched or appear redundant}

### Upcoming Deadlines
{From Business_Goals.md active projects:}
- {Project}: due {date} ({N days} away)

---

## Errors & Alerts This Week

{List any logged errors from /Logs/:}
- [{date}] {error description}

{If none: ✅ No errors logged this week.}

---

## Recommended Actions for You

1. {Most important action — be specific}
2. {Second action}
3. {Third action}

---

*Generated by AI Employee v0.1 — Silver Tier*
*Next briefing: {next Monday date}*
```

### 4. Update Dashboard.md

Add to **Recent Activity**:
```
- [{timestamp}] Weekly briefing generated: {YYYY-MM-DD}_Monday_Briefing.md
```

Update **Notes** section to reference the new briefing.

### 5. Write Log Entry

Append to `/Logs/{YYYY-MM-DD}.json`:
```json
{
  "timestamp": "{ISO}",
  "action_type": "weekly_briefing_generated",
  "actor": "claude_code",
  "briefing_file": "{YYYY-MM-DD}_Monday_Briefing.md",
  "period_start": "{date}",
  "period_end": "{date}",
  "result": "success"
}
```

## Rules

- Always base data on actual vault files — never fabricate numbers
- If a KPI is missing (e.g., MTD revenue not tracked), note "data not available" rather than inventing
- Flag any KPI that has crossed an alert threshold from Business_Goals.md
- Proactive suggestions must reference actual data — not generic advice
- Create `/Briefings/` folder if it doesn't exist
